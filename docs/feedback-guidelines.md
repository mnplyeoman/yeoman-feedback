# Feedback Guidelines

## Хороший фидбек

Хороший фидбек помогает понять:

- где именно возникла проблема;
- что ожидал пользователь;
- какой был фактический опыт;
- почему это важно.

## Полезный формат

Можно использовать простую структуру:

1. Контекст.
2. Наблюдение.
3. Влияние.
4. Предложение.

## Пример

Контекст: создавал новый сценарий настройки.

Наблюдение: не хватило пояснения, что поле обязательно.

Влияние: пришлось несколько раз возвращаться назад и проверять форму.

Предложение: добавить inline-подсказку и текст ошибки до отправки формы.

## Чего лучше избегать

- слишком общих формулировок без примеров;
- нескольких несвязанных тем в одном issue;
- эмоциональных оценок без описания сценария;
- публикации чувствительных данных.
