# yeoman-feedback

Репозиторий для сбора обратной связи по проекту Monopoly - yeoman.

Здесь мы собираем:

- баг-репорты;
- предложения по улучшению;
- UX-фидбек;
- замечания по документации;
- общие идеи по развитию проекта.

## Цель репозитория

Сделать единое, понятное место, где команда и пользователи могут оставлять обратную связь по Monopoly - yeoman, а также отслеживать, что уже обсуждается и что принято в работу.

## Как оставить обратную связь

1. Откройте раздел `Issues`.
2. Выберите подходящий шаблон.
3. Опишите ситуацию как можно конкретнее.
4. Если похожая тема уже существует, лучше продолжить её, а не создавать дубликат.

## Какие типы обращений есть

- `Bug report`:
  если что-то работает не так, как ожидается.
- `Feedback`:
  если хотите поделиться впечатлением, болями, наблюдениями или контекстом использования.
- `Improvement`:
  если есть конкретное предложение по улучшению интерфейса, логики, сценариев или процессов.

## Что помогает нам быстрее обработать фидбек

- понятный заголовок;
- шаги воспроизведения;
- ожидаемый и фактический результат;
- скриншоты или примеры, если они есть;
- контекст: версия, окружение, сценарий использования, частота проблемы;
- описание влияния на пользователя или команду.

## Правила работы

- Будьте уважительны к другим участникам.
- Один issue должен описывать одну проблему или одну идею.
- Критикуйте решение, а не людей.
- Избегайте общих формулировок без примеров и контекста.
- Не публикуйте секреты, токены, пароли и персональные данные.
